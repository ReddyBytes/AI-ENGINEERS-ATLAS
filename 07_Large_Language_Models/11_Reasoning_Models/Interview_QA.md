# Reasoning Models — Interview Q&A

---

## Beginner

**Q1: What is a reasoning model and how does it differ from a standard LLM?**

A reasoning model generates explicit chain-of-thought reasoning steps (thinking tokens) before producing a final answer. Standard LLMs generate tokens left-to-right without backtracking. Reasoning models spend compute *thinking through* the problem first, then commit to an answer — similar to how a human might work through a math problem on scratch paper before writing the final answer.

---

**Q2: What are "thinking tokens"?**

Thinking tokens are internal reasoning steps generated by the model before its final response. In some implementations (Claude Extended Thinking, DeepSeek-R1) they are visible to the user. In others (OpenAI o1) only a summary is shown. Thinking tokens are billed separately and contribute to the model's reasoning quality but are not part of the visible response.

---

**Q3: Name three reasoning models and their makers.**

- **o1 / o3** — OpenAI
- **DeepSeek-R1** — DeepSeek (open source)
- **Claude with Extended Thinking** — Anthropic (claude-sonnet-4-6 and above)
- **Gemini 2.0 Flash Thinking** — Google

## Intermediate

**Q4: When would you choose a reasoning model over a standard LLM?**

Use reasoning models for:
- Multi-step mathematical proofs or calculations
- Complex algorithm design or debugging
- Scientific reasoning requiring logical chains
- Planning problems with many constraints

Avoid reasoning models for:
- Simple Q&A or summarization (overkill — wasted cost + latency)
- High-volume, latency-sensitive applications (chatbots, autocomplete)
- Creative writing where reasoning doesn't add value

---

**Q5: What is a Process Reward Model (PRM) and why does it matter for reasoning?**

A standard reward model scores the final answer (correct/incorrect). A **Process Reward Model** scores each intermediate reasoning step — rewarding correct reasoning even if the final answer is eventually wrong. This trains the model to develop good reasoning habits rather than just pattern-matching to correct-looking answers. PRMs are the key training signal that separates reasoning models from standard RLHF-trained models.

---

**Q6: How does test-time compute scaling work?**

In standard LLMs, output quality is fixed by model size — you can't improve it at inference time. Reasoning models support **test-time compute scaling**: more thinking tokens = better accuracy (up to a point, following a logarithmic curve). This means you can trade cost for quality on hard problems by increasing the thinking budget, without retraining the model.

---

**Q7: What API differences should you be aware of when switching from GPT-4 to o1?**

- **No system prompt**: o1 does not support a system message — all instructions go in the user turn
- **No temperature**: o1 doesn't accept temperature, top_p, or other sampling parameters
- **Higher max_tokens needed**: responses include reasoning traces that consume tokens
- **Longer latency**: expect 10–60s response times vs milliseconds for standard models
- **Different pricing**: o1 is significantly more expensive per token than GPT-4o

## Advanced

**Q8: How does DeepSeek-R1 achieve reasoning capability and why is it significant?**

DeepSeek-R1 was trained using large-scale reinforcement learning with process reward signals — similar to OpenAI's approach for o1 but released open source. Its significance:
1. **Open source**: anyone can inspect, fine-tune, or self-host it
2. **Competitive performance**: matches o1 on most reasoning benchmarks
3. **Distilled versions**: R1-Distill-Qwen-7B and R1-Distill-Llama-8B bring reasoning to small models that run locally via Ollama
4. **Full CoT visible**: unlike o1, DeepSeek-R1 shows the complete chain-of-thought in `<think>` tags

---

**Q9: How do you configure Claude's Extended Thinking and what tradeoffs should you consider?**

```python
response = client.messages.create(
    model="claude-sonnet-4-6",
    max_tokens=16000,          # must be > budget_tokens
    thinking={
        "type": "enabled",
        "budget_tokens": 10000  # how much to spend thinking
    },
    messages=[{"role": "user", "content": "..."}]
)
```

Tradeoffs:
- **Higher budget** → better accuracy on hard problems, higher cost, longer latency
- **Lower budget** → faster, cheaper, but may not solve complex problems
- **Streaming**: thinking tokens stream separately from response tokens
- **Cost**: thinking tokens are billed at the same rate as output tokens — a 10k thinking budget costs significantly more

---

**Q10: A user reports your AI coding assistant gives wrong answers on complex refactoring tasks. How do you debug and fix this?**

1. **Classify the failure**: is it a reasoning failure (wrong logic) or a knowledge failure (wrong API usage)?
2. **If reasoning failure**: switch to a reasoning model (o1, Claude Extended Thinking, DeepSeek-R1)
3. **Set appropriate budget**: for complex refactoring, use 8,000–15,000 thinking tokens
4. **Provide structured context**: reasoning models benefit from clear problem decomposition in the prompt
5. **Evaluate**: compare accuracy on a held-out set of complex refactoring tasks before/after the switch
6. **Monitor cost/latency**: reasoning models cost 3–10× more — assess if the accuracy gain justifies it
7. **Consider hybrid**: use reasoning model only for complex tasks, standard model for simple completions
