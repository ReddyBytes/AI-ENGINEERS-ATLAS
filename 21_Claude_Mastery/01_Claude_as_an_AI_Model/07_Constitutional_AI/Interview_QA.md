# Constitutional AI — Interview Q&A

## Beginner

**Q1: What is Constitutional AI and why did Anthropic develop it?**

<details>
<summary>💡 Show Answer</summary>

Constitutional AI (CAI) is Anthropic's training methodology for building safer AI systems. Instead of relying entirely on human annotators to label which responses are harmful, CAI uses the model itself to critique and revise its outputs against a written set of principles — the "constitution."

Anthropic developed it to solve a fundamental bottleneck in RLHF: human annotation throughput. To train a large model on harmlessness, you need millions of preference comparisons. Hiring humans to evaluate that many examples is slow, expensive, and inconsistent (different labelers have different values).

CAI replaces much of the harmlessness annotation with AI-generated feedback. The model applies the constitution to generate critique-revision pairs, which become training data. This scales without proportional human labor.

The secondary benefit: the written constitution makes the model's value system auditable. Anyone can read the principles and understand what the model was trained to do — a transparency advantage over implicit labeler preferences.

</details>

**Q2: What is a "constitution" in Constitutional AI?**

<details>
<summary>💡 Show Answer</summary>

The constitution is a document containing principles that the model uses to evaluate its own outputs. These are written statements like:
- "Choose the response that is least likely to provide assistance with illegal activities"
- "Choose the response that is most respectful of human dignity"
- "Choose the response that a thoughtful, senior Anthropic employee would be proud of"

The constitution draws from multiple ethical frameworks: UN human rights principles, common-sense ethics, and Anthropic's own safety guidelines. It is not a list of banned words or topics — it is a set of values that the model applies contextually.

The key insight: making the principles explicit means you can read them, critique them, update them, and trace safety behaviors back to specific principles. This is more interpretable than implicit labeler preferences encoded in a reward model.

</details>

**Q3: How does the CAI self-critique loop work in practice?**

<details>
<summary>💡 Show Answer</summary>

The self-critique loop has two steps — critique and revision:

1. **Generate**: The model produces a response to a prompt (often an adversarial/harmful prompt generated by red-teaming)

2. **Critique**: The model is asked "Does this response violate [principle X]? How could it be improved?" It identifies the problem.

3. **Revise**: The model rewrites the response to comply with the principle.

4. **Repeat**: The critique-revise cycle runs for several principles in sequence, with each revision building on the previous.

5. **Train**: The final (prompt, revised_response) pair becomes supervised training data.

This process can generate millions of training examples by running the model on diverse prompts with different constitutional principles as the critique lens.

</details>

## Intermediate

**Q4: How does CAI differ from standard RLHF in terms of what's being replaced?**

<details>
<summary>💡 Show Answer</summary>

CAI and RLHF are not alternatives — CAI extends RLHF by replacing specific components:

In standard RLHF:
- Stage 1 (SFT): Human-written demonstrations → train the model
- Stage 2 (RM): Human preference rankings → train reward model  
- Stage 3 (RL): Reward model + PPO → align the model

CAI modifies Stage 1 and 2 for harmlessness specifically:
- Stage 1 becomes SL-CAI: AI self-critique replaces some human demonstrations
- Stage 2 becomes RLAIF: AI preference judgments replace some human rankings

What is NOT replaced: human feedback for helpfulness, capability evaluation, and general quality judgment. Anthropic still uses human annotation — CAI reduces the human burden specifically for harmlessness annotation.

</details>

**Q5: Why is CAI described as "making the value system auditable"?**

<details>
<summary>💡 Show Answer</summary>

In standard RLHF, the model's values are implicitly encoded in the reward model. The reward model learned from human preferences, but those preferences were never explicitly written down. You can't read the reward model to understand what values it represents.

With Constitutional AI, the values are a written document. You can:
- Read the principles and understand what the model was trained to prioritize
- Identify specific principles that might be causing problematic behavior
- Add, remove, or modify principles and observe the effect on training
- Show the constitution to external parties for review and critique
- Trace a specific model behavior back to the principle that trained it

This is analogous to the difference between "train a classifier on labeled data" (implicit) and "program explicit rules" (explicit). CAI is more explicit than standard RLHF, though still not fully interpretable.

</details>

**Q6: What are the limitations of Constitutional AI?**

<details>
<summary>💡 Show Answer</summary>

CAI has several important limitations:

1. **Constitution quality ceiling**: The model can only apply the principles as well as it understands them. Subtle principles that require nuanced judgment may not be applied correctly, especially early in training.

2. **Bootstrap problem**: The first round of self-critique is done by the base model, which hasn't been safety-trained yet. The quality of early critiques is limited by base model capability.

3. **AI preference ≠ human preference**: AI-generated preference rankings may systematically differ from human preferences in ways that aren't obvious. Research suggests AI feedback can introduce biases.

4. **Scope**: CAI is primarily designed for harmlessness. Helpfulness training still requires human judgment.

5. **Specification problem**: Writing a good constitution is hard. Principles that sound reasonable can have unintended consequences at scale. A principle like "avoid all potentially dangerous content" could cause excessive refusals.

6. **Gaming**: A model trained to critique according to the constitution may learn to satisfy the constitution's letter without its spirit.

</details>

## Advanced

**Q7: How does RLAIF (RL from AI Feedback) work in the CAI framework, and what are its properties compared to RLHF?**

<details>
<summary>💡 Show Answer</summary>

RLAIF uses AI-generated preference rankings instead of human-generated ones to train the reward model in Stage 2 of RLHF.

Process:
1. Generate response pairs (A, B) for the same prompt using the current model
2. Feed each response to the AI evaluator with a constitutional principle: "Which response better follows principle X?"
3. The AI evaluator outputs a preference: A > B or B > A
4. Use these preferences to train the reward model
5. Use the reward model in PPO/DPO as usual

Properties compared to RLHF:

Advantages:
- Scale: generate millions of preference pairs cheaply
- Consistency: same principle applied the same way every time
- Speed: no human scheduling or quality control overhead
- Coverage: can densely evaluate edge cases that human labelers rarely encounter

Limitations:
- The AI evaluator has its own biases (e.g., may prefer longer or more verbose responses)
- The evaluator's "values" reflect its training, not independent human values
- Agreement with constitutional principles doesn't guarantee agreement with diverse human preferences
- Requires the evaluator to be capable enough to reliably apply principles — weak evaluators give noisy signals

In practice, RLAIF produces high-quality preference data for harmlessness but is supplemented with human feedback for nuanced helpfulness judgments.

</details>

**Q8: How does red-teaming feed into the Constitutional AI training loop, and why is this described as a "virtuous cycle"?**

<details>
<summary>💡 Show Answer</summary>

Red-teaming is the practice of systematically searching for prompts that cause a model to produce harmful outputs. In the CAI context, red-teaming does more than just identify problems — it directly feeds the training data pipeline.

The cycle:
1. **Red-team**: Use humans or automated systems to find prompts that cause harmful outputs
2. **Generate**: Run these harmful prompts through the model to get harmful responses
3. **CAI loop**: Apply constitutional critique-revise to those harmful responses
4. **Train**: Add (harmful_prompt, safe_revised_response) pairs to training data
5. **Evaluate**: Test the newly trained model — red-team again with harder prompts
6. **Repeat**

Why it's "virtuous": each iteration closes the specific failure modes discovered in the previous round. Unlike a static safety filter that blocks known patterns, this approach generates new training data for each new failure type discovered.

The virtuous cycle has practical limits — sufficiently creative adversaries can always find new jailbreaks. But the attack difficulty increases with each training cycle, and the model becomes more robust at each iteration.

</details>

**Q9: How would you implement a simplified Constitutional AI critique-revise system?**

<details>
<summary>💡 Show Answer</summary>

Here's the basic architecture using the Claude API:

```python
import anthropic

client = anthropic.Anthropic()
CONSTITUTION = [
    "Does this response assist with any clearly illegal activities?",
    "Is this response respectful to all people regardless of their background?",
    "Could this response be used to cause physical harm to someone?",
]

def apply_constitution(prompt: str, response: str, model="claude-haiku-4-5") -> str:
    """Apply constitutional critique-revise to a response."""
    current_response = response
    
    for principle in CONSTITUTION:
        # Critique
        critique_prompt = f"""Original prompt: {prompt}

Response: {current_response}

Constitutional check: {principle}

First, identify whether the response violates this principle.
Then, if it does, rewrite the response to comply.
If it doesn't violate the principle, output "COMPLIANT" only."""
        
        result = client.messages.create(
            model=model,
            max_tokens=1024,
            messages=[{"role": "user", "content": critique_prompt}]
        ).content[0].text
        
        if "COMPLIANT" not in result.upper():
            current_response = result  # use the revised response
    
    return current_response
```

This simplified example shows the core pattern. Production CAI uses more sophisticated prompting, iterative refinement, and trains on the pairs at scale.

</details>
