# Memory System — Cheatsheet

## Memory Types at a Glance

| Type | What it holds | Where it lives | Who writes it |
|------|--------------|---------------|---------------|
| Project memory | Codebase facts, patterns, quirks | `.claude/memory/MEMORY.md` | Claude (auto) + you |
| Global memory | Cross-project preferences | `~/.claude/projects/<hash>/memory/` | Claude (auto) |
| User/feedback | Preferences, corrections | Same as project | Claude (auto) |
| Reference | Architecture docs, decisions | Linked from MEMORY.md | You + Claude |

---

## MEMORY.md Structure Template

```markdown
# Project Memory — [Project Name]
Last updated: YYYY-MM-DD

## Architecture
- **[Component]:** [fact] — [why it matters / where to find it]

## Key Patterns
- **[Pattern name]:** [description]

## Test Setup
- **Run tests:** `pytest tests/ -v`
- **[Fixture]:** [description]

## User Preferences
- [Preference 1]
- [Preference 2]

## Known Quirks / Tech Debt
- [quirk 1]
- [quirk 2]
```

---

## Triggering Auto-Memory

Claude saves automatically when it discovers:
- Architecture patterns not obvious from code
- Undocumented behaviors
- Setup quirks
- User preferences stated in conversation

Explicit triggers:
```
> Remember that X
> Save to memory: Y
> Make a note that Z
```

---

## Reading Memory Explicitly

```
> What do you know about the database setup?
> Show me everything you've saved for this project
> What are the known quirks in this codebase?
```

---

## What to Save vs Skip

| Save | Skip |
|------|------|
| Architecture decisions | Step-by-step task progress |
| Undocumented behaviors | Things obvious from reading code |
| User preferences | Temporary state |
| Known bugs / tech debt | Highly volatile facts |
| Tricky test setup | Personal information |

---

## File Locations

```
~/.claude/
└── projects/
    └── <project-hash>/
        └── memory/
            ├── MEMORY.md        ← index
            ├── architecture.md  ← detail file
            └── patterns.md      ← detail file

<project>/
└── .claude/
    └── memory/
        └── MEMORY.md            ← project-scoped index
```

---

## Memory vs CLAUDE.md

| CLAUDE.md | MEMORY.md |
|-----------|-----------|
| Instructions (what to do) | Facts (what is true) |
| Written by you | Auto-generated by Claude |
| Static, version-controlled | Dynamic, grows over time |
| Project rules and conventions | Discovered architecture and quirks |
