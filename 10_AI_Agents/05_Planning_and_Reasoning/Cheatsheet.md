# Planning and Reasoning — Cheatsheet

**One-liner:** Planning breaks a complex goal into a sequence of manageable sub-tasks before execution — this dramatically improves reliability on multi-step tasks compared to letting the agent figure it out on the fly.

---

## Key Terms

| Term | What it means |
|---|---|
| **Planner** | An LLM call (or agent) that takes a high-level goal and produces a list of tasks |
| **Executor** | An agent that takes individual tasks from the planner and executes them one at a time |
| **Task list** | The sequence of concrete steps generated by the planner |
| **Chain-of-Thought (CoT)** | Prompting the LLM to write out reasoning steps before giving an answer — improves accuracy |
| **Plan-and-Execute** | Architecture: planner creates task list, executor runs each task |
| **Tree of Thoughts (ToT)** | Exploring multiple plan branches simultaneously — finds better solutions but slower |
| **BabyAGI** | An autonomous loop: complete task → generate next tasks → prioritize → repeat |
| **Decomposition** | Breaking a complex goal into smaller, independently executable steps |
| **Replanning** | When an executor hits a dead end, triggering the planner to revise the task list |
| **Task dependency** | When task B requires the output of task A before it can run |

---

## Planning Approaches Compared

| Approach | Complexity | Best for | Weakness |
|---|---|---|---|
| Chain-of-Thought | Low | Moderate complexity | Still one agent, can drift |
| Plan-and-Execute | Medium | Structured tasks with known shape | Plan may be wrong upfront |
| Tree of Thoughts | High | Creative, open-ended problems | Much slower, expensive |
| BabyAGI-style | High | Open-ended research/exploration | Can generate too many tasks |

## Plan-and-Execute in Code (Pseudocode)

```python
# Step 1: Generate the plan
plan = planner_llm.invoke(
    f"Create a step-by-step task list to accomplish: {goal}"
)

# Step 2: Execute each task
results = []
for task in plan.tasks:
    result = executor_agent.run(
        task=task,
        context=results  # Previous results as context
    )
    results.append(result)

# Step 3: Synthesize
final_answer = synthesizer_llm.invoke(
    f"Goal: {goal}\nResults: {results}\nWrite the final response."
)
```

---

## When to Use Planning

**Use planning when:**
- The task has 5+ steps
- Steps need to be done in a specific order
- The task requires research before synthesis
- You want to track progress explicitly
- The task spans multiple domains (research + writing + analysis)

**Skip explicit planning when:**
- Simple 1-3 step tasks
- The task is purely reactive (answer questions as they come)
- Latency is critical (planning adds extra LLM calls)

---

## Golden Rules

1. **Decompose first, execute second.** Never let the agent figure out the plan as it goes on complex tasks.

2. **Each task should be independently executable.** "Write the executive summary" is good. "Do the business stuff" is not.

3. **Tasks should have clear completion criteria.** How does the executor know when a task is done?

4. **Allow replanning.** If a task fails or reveals new information, the planner should revise the remaining tasks.

5. **Pass results forward.** Each task's output should be available to subsequent tasks as context.

6. **Keep tasks small.** If a task takes more than 5 steps to execute, break it down further.
