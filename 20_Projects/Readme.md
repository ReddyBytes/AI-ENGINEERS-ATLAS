## How Each Project Is Structured

Every project folder contains:

| File | What It Is |
|---|---|
| `Project_Guide.md` | What you'll build, why it matters, what you'll learn |
| `Step_by_Step.md` | Numbered build instructions from setup to working system |
| `Starter_Code.md` | Runnable Python with `# TODO:` markers for you to complete |
| `Architecture_Blueprint.md` | System diagram + component breakdown |

---

## The Golden Rule

**Don't copy-paste the starter code and call it done.** The `# TODO:` sections are where the learning happens. Struggle with them. Check the Theory.md if you're stuck. The discomfort is the point.
