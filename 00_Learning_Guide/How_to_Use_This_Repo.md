# How to Use This Repo

This repo is a structured, story-first AI knowledge base. Here's how to get the most out of it.

---

## The 3-File System

Every topic has exactly 3 core files:

| File | When to read it |
|---|---|
| `Theory.md` | First read — the full explanation with story, diagrams, and examples |
| `Cheatsheet.md` | Before interviews or when you need a quick refresher |
| `Interview_QA.md` | After understanding the topic — test yourself |

Some topics also have bonus files: `Code_Example.md`, `Math_Intuition.md`, `Architecture_Deep_Dive.md`, etc. Read those when you want to go deeper.

---

## How to Read Theory.md

Each Theory.md follows this structure:

1. **The Story** — A real-world analogy. Read it slowly. Let it land.
2. **What is X?** — The formal definition after the intuition.
3. **How It Works** — Step-by-step with a Mermaid diagram.
4. **Examples** — Where you'll see this in the real world.
5. **Mistakes** — What trips people up.
6. **Connections** — What to read next.
7. **✅ / 🔨 / ➡️** — Summary, hands-on task, next step.

---

## Recommended Workflow Per Topic

```
Read Theory.md
    ↓
Try the 🔨 Build task
    ↓
Skim Cheatsheet.md
    ↓
Test yourself with Interview_QA.md
    ↓
Move to the next topic (follow the ➡️ link)
```
